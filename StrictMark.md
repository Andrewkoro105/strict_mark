Основная идея разделение структурированого текста и стиля. С возможностью экспорта в word с указанным стилем и и с результатами всех расчётов (но с полным определением, гарантируя переносимость файла). И сисиемой расширения похволяющая выпонять блоки разного кода, по надобности объеденяя их в одно пространство. Результатом таких вычислений всегда является либо текст либо картинка.
Система стиля приэтом может не только указывать параметры форматирования блоков, но менять их с помощью lua и настраивать ссылки на элементы втоматически следя за нумерацией.

# Структурированный текст
Структура текста описывается модифицированным markdown-ном без поддержки html.
Все структурные элементы должны быть разделены пустыми строками (кроме имён и параметров так как они являются подобием модификаторов)

# Структурные элементы
## Имена
Чтобы сослаться на какой либо элемент нужно его проименовать. Это необходимо ибо нумерация принятия в таких случаях может съехать.
Для именования элемента над ним нужно приписать `[elemet name]` а чтобы сослаться нужно указать `@(elemet name)`. Если элемент имеет именованные под элементы то они указываются через `:` например @(elemet name 1):(elemet name 1.1).
У них есть булевый параметр `namespace` Которое превращает его в пространство имён для дочерних объектов, поумолчанию оно false но у списков и таблиц оно true

## Параметры
Некоторые структурные элементы имеют параметры, они указываются с помощью `{}`, и путся в формате `name = value`, но будевые значения могут опускать явное описание значения если являются необязательными и их значение поумолчанию false, например: `{bool_param, int_param = 6}`.

## Комментарии
Всё после `%` коментарий и не имеет значение.

## Заголовки
Заголовки помечаются с помощью `#`, чем больше их число тем меньше уровень заголовка.

## Абзацы
Блок текста, для большей читаемости чистого файла в редакторах без авто переноса. Переносы без пустых строк заменяется на пробелы.
Абзацы могут быть разного типа и для его изменения есть параметр `type`, который может принимать значения:
- `text` - обычный абзац текста, значение поумолчанию;
- `quote` - форматируется как цитата;
- `footnote` - форматируется как сноска (большинство стилей должно вставлять её в конец страницы на которой она упомянута).

## Перечисления 
С помощью `-` можно создавать перечисления. Для управления внешним видом есть параметр `{type}`, у него есть несколько значений:
- `number` - в качестве маркеров используются числа, это значение поумолчанию;
- `mark` - в качестве маркеров используется константный маркер ввбраный стилем;
- `char` - в качестве маркеров используются буквы;
- `bibliography` - весь список и ссылки на него форматируется как библиографический список;
- `definitions` - весь список и ссылки на него форматируется как список определений.
Чтобы сослаться на конкретный элемент можно использовать его номер или имя если указать (второй вариант более рекомендуем). Текст внутри выравнивания с помощью пробелов к своему началу и пишется как обычный текст

## Форматирование
Текст можно сделать жирным `**жирный текст**`, курсивным `*курсивный текст*` , подчёркнутым `__подчёркнутый текст__`,  зачёркнутым `~~зачёркнутый текст~~` и неделимый текст с помощью \`.

## Формулы
В одинарных `$` можно будет писать внутристрочные формулы на LaTeX, а в двойных те что будут вынесены из текста и пронумерованны многими стилями.

## Иные файлы
Иные файлы вставляются похожим образом но делятся на 2 типа, вставка страницей и вставка ресурсом.
### Вставка страницей
Вставляет изображения и pdf как страницы `(path)`

### Вставка ресурсом
Вставляет изображения добавляя к ним подписи, если несколько таких изображений указаны подряд без пустых строк, то они будут объединены в группу.
```
(path1.png)"
    text1
"
(path2.png)"
    text2
"
```

## Таблицы
 Таблицы можно создавать похожим на markdown способо, но он расширен дабы дать возможность объединять ячейки и указывать как в ячейках должен распологатся текст.

 К сожалению в документах таблица нужно оченб много функций но обычно они не используются все сзразу так что оно не должно выглядеть супер сложно.

 Между каждой строкой идёт разделение с чётким указанием сколько ячеек в следующей и как в них расположены данные. Всё такие разделители должны чётко совпадать по числу символов а разделители колонок должны стоять в одном столбце чтобы указывать край одной колонки.
 Колонки разделяются символами указвающими как в ячейке снизу справа от разделителя располагается текст. Доступны 3 варианта:

 - `|` - поцентру
 - `^` - сверху
 - `v` - снизу

 Около разделителей могут стоять `:` они указывают к какому краю равнять текст в ячейке подними, если их нет то текст располагается по середине (ставить двоеточия на оба края запрещено).
 Оставшаяся место между разделителями должно быть заполнено `-` или `=` если строка сверху это шапка.
 
 Если нужно объединить ячейки по вертикали то в промежуточных слоях нужно использовать только `|` как разделитель и ставить пробелы до следующего разделителя.

Строки разделяются ` | ` который должен строго совпадать с разделителями сверху. Текст внутри может быть многострочным, к нему применяются те-же системы что и ко всему файлу.

Также клюбой колонке можно дать имя и тип чтобы к ней можно было обращаться из расширений. Для этого нужно в добавить к разделю строк который форматирует верхную ячейку новую строку в которой левый разделитель начальной ячейки будет заменён на `s` и через `-` будет указанны `[name]type`, оставшееся простанство до следующего разделитель следует заполнить `-` или пробелами если ячейки объеденены. Для указания конца колонки оспользуется дополнительный строчный разделитедль после последней яцейки, в нём у колонки которую нужно заверщить левый разделитель заменяется `e`, а остальные остаются `|` пространство между ними должно быть заполнено `-` или пробелами если ячейки объеденены. Приэтом все зраделители этой колонки должны состоять тольоко из `|` и `-` ибо форматирование ячеек наследуюется из первой.

После таблицы в кавычках идёт описание таблицы, тест внутри имеет обязательный отступ если он пногострочный.

В примере ниже первая буква указывает на то как будет выравнен текст по вертикали, вторая по горизонтали.
```
[table name]
^:-------------|------v-----:|
| вл           | цц   | нп   |
|---------------------|      |
| цц                  |      |
|-------------:^-------------|
| цп           | вц          |
|=============:|=============|
s-[name]type---|-------------|
| цп           | цц          |
|--------------|:------------|
| цп           | цл          |
e--------------|             |
v-------------:|             |
| нп           |             |
"
    многострчное описание
"
```

## Блоки кода
Блоки кода это основной способ расширения возможностей редактора.
В специальном файле можно прописать какая программа исполнит этот код и как извлеч её результат. Но в конце концов результатом всегда будет текст в формате этого редактора и набор ресурсов (по типу изображений) которые этот текстобязан вставить.

Блоки кода прописываются как и в markdown но вместо языка указывается название расширения в котором настроено исполнение этого кода.

Например так будут реализованны диаграммы, авто расчёт математики, LaTeX.
